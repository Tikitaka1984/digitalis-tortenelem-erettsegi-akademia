# 03 – H5P Standards

## Mire szolgál?

A H5P Interactive Book implementációk szerkezeti, kompatibilitási, lokalizációs, UX-, accessibility- és csomagolási követelményeit rögzíti.

A hivatalos fejlesztési módszer `Work + automatikus QA + GitHub Pages`. A Lumi Desktop kizárólag prototípus-, kompatibilitási és hibakeresési eszköz, nem hivatalos build-, QA- vagy publikációs környezet.

## Milyen fájlok kerülnek ide?

- támogatott H5P-library mátrixok;
- komponenshasználati és konfigurációs szabványok;
- lokalizációs és visszajelzési konvenciók;
- csomagmanifest-, függőség- és buildkövetelmények;
- H5P-validációs sémák és ellenőrzőlisták.

## Mi nem kerülhet ide?

- kész modulcsomag;
- modul-specifikus történelmi szöveg;
- licenc nélküli H5P-library vagy módosított külső bináris;
- konkrét tanulói feladat példánya.

## Workflow-kapcsolat

Az Instructional Design és a Build Guide jóváhagyása után irányítja az implementációt, majd az automatikus és kézi H5P QA ellenőrzési alapja.
